---
name: agentic-tdd
description: test-driven-development 의 강화 버전.
---

tdd 를 진행할 때 human-in-the-loop 를 강조해서,
각 싸이클의 RED-GREEN-REFACTOR 단계를
RED-GREEN-REVIEW 으로 수정할꺼야.



각 RED,GREEN,REVIEW 단계에서 중요한 부분은
- RED 에서 테스트 작성계획에 대한 Plan.md 를 만들어서 사람한테 검토를 맡고 테스트를 작성해야한다.
- REVIEW 에서 GREEN 단계의 코드를 검토후 다음 싸이클을 진행할것이야.



절대로 RED,REVIEW 단계를 사람의 검토 없이 지나가면 안돼.
RED,REVIEW 단계가 끝날때 검토해달라고 말해줘
