==============================================================
TC02 | 재고 부족 시 주문 접수 → PRODUCING (생산 라인 등록)
==============================================================
목적: 주문 수량이 현재 재고보다 많을 때 자동으로 생산 라인에 등록되는지 확인

초기 재고 (samples.json 기준)
  S003 | GaN 에피택시 웨이퍼 | 재고 15개

--------------------------------------------------------------
STEP 1. 프로그램 실행
--------------------------------------------------------------
  명령어: ./run.sh

--------------------------------------------------------------
STEP 2. 주문 접수 (재고 초과 수량)
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 1 (주문 접수)
  입력:
    시료 ID : S003
    고객명  : 팹리스A
    수량    : 30   ← 현재 재고(15)보다 많음

  기대 결과:
    - "주문 접수 완료" 메시지 및 주문 ID 표시
    - 상태: RESERVED

--------------------------------------------------------------
STEP 3. 주문 승인
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 2 (주문 승인)
  위 주문 ID 입력

  기대 결과:
    - "주문 승인 - 생산 대기 중 (PRODUCING)" 메시지 출력
    - 상태가 CONFIRMED가 아닌 PRODUCING

--------------------------------------------------------------
STEP 4. 생산 큐 확인
--------------------------------------------------------------
  메인 메뉴 → 4 (생산 라인 관리) → 1 (생산 큐 조회)

  기대 결과:
    - 위 주문이 생산 큐에 등록되어 있음
    - 수량 30, 상태 PRODUCING 표시

--------------------------------------------------------------
판정 기준
  PASS: 주문 상태 PRODUCING / 생산 큐에 항목 존재
  FAIL: 재고 부족인데도 CONFIRMED 처리된 경우
