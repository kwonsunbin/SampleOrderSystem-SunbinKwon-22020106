==============================================================
TC03 | 주문 거절 → REJECTED 상태 전환
==============================================================
목적: RESERVED 상태 주문을 거절하면 REJECTED로 전환되고
      이후 모니터링에서 제외되는지 확인

--------------------------------------------------------------
STEP 1. 프로그램 실행
--------------------------------------------------------------
  명령어: ./run.sh

--------------------------------------------------------------
STEP 2. 주문 접수
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 1 (주문 접수)
  입력:
    시료 ID : S001
    고객명  : 거절테스트
    수량    : 5

  주문 ID 메모 (화면에 표시된 UUID)

--------------------------------------------------------------
STEP 3. 주문 거절
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 3 (주문 거절)
  위 주문 ID 입력

  기대 결과:
    - "주문 거절 완료 (REJECTED)" 메시지 출력

--------------------------------------------------------------
STEP 4. 승인 목록에서 제외 확인
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 2 (주문 승인)

  기대 결과:
    - 거절한 주문이 목록에 더 이상 표시되지 않음
    - (다른 RESERVED 주문이 없으면 "대기 중인 주문이 없습니다." 출력)

--------------------------------------------------------------
STEP 5. 모니터링에서 REJECTED 제외 확인
--------------------------------------------------------------
  메인 메뉴 → 3 (모니터링) → 주문 현황 조회

  기대 결과:
    - REJECTED 항목이 모니터링 집계에 포함되지 않음

--------------------------------------------------------------
판정 기준
  PASS: 상태 REJECTED / 승인 목록 미표시 / 모니터링 제외
  FAIL: 거절 후에도 승인 목록에 남아 있는 경우
