==============================================================
TC10 | 예외 및 엣지 케이스 — 잘못된 입력 / 경계 조건
==============================================================
목적: 시스템이 비정상 입력에 대해 크래시 없이 적절한 오류 메시지를
      출력하고 정상 상태를 유지하는지 확인

--------------------------------------------------------------
케이스 A: 존재하지 않는 시료 ID로 주문 접수
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 1 (주문 접수)
  입력:
    시료 ID : S999   ← 존재하지 않는 ID
    고객명  : 테스트
    수량    : 1

  기대 결과:
    - 오류 메시지 출력 (예: "해당 시료를 찾을 수 없습니다")
    - 프로그램 종료 없이 메뉴로 복귀

--------------------------------------------------------------
케이스 B: 수량 0 또는 음수로 주문 접수
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 1 (주문 접수)
  입력:
    시료 ID : S001
    고객명  : 테스트
    수량    : 0   (또는 -1)

  기대 결과:
    - 오류 메시지 출력
    - 주문 생성 없이 메뉴 복귀

--------------------------------------------------------------
케이스 C: 빈 생산 큐에서 처리 시도
--------------------------------------------------------------
  (생산 큐가 비어있는 상태에서)
  메인 메뉴 → 4 (생산 라인 관리) → 2 (다음 주문 처리)

  기대 결과:
    - "생산 큐가 비어 있습니다" 또는 유사 오류 메시지
    - 프로그램 크래시 없이 메뉴 복귀

--------------------------------------------------------------
케이스 D: 존재하지 않는 주문 ID로 승인 시도
--------------------------------------------------------------
  메인 메뉴 → 2 (주문 관리) → 2 (주문 승인)
  주문 목록에서 실제 존재하지 않는 ID 직접 입력:
    예: "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx"

  기대 결과:
    - 오류 메시지 출력
    - 프로그램 크래시 없이 메뉴 복귀

--------------------------------------------------------------
케이스 E: 이미 CONFIRMED된 주문을 다시 승인 시도
--------------------------------------------------------------
  사전 조건: CONFIRMED 상태 주문이 있어야 함

  메인 메뉴 → 2 (주문 관리) → 2 (주문 승인)
  → CONFIRMED 주문은 목록에 표시되지 않아야 함
     (RESERVED 목록에만 표시되므로 자동으로 방지됨)

  기대 결과:
    - CONFIRMED 주문이 승인 목록에 없음
    - 이중 승인 자체가 UI에서 불가능

--------------------------------------------------------------
케이스 F: 메뉴 숫자 외 잘못된 입력
--------------------------------------------------------------
  메인 메뉴에서 숫자 외 문자 입력: "abc"

  기대 결과:
    - "잘못된 선택입니다" 또는 유사 메시지
    - 프로그램 종료 없이 계속 동작

--------------------------------------------------------------
판정 기준
  PASS: 모든 케이스에서 크래시 없이 오류 메시지 후 정상 복귀
  FAIL: 프로그램이 예외로 종료되거나 잘못된 데이터가 저장된 경우
