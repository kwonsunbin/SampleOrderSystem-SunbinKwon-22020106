==============================================================
TC06 | 모니터링 — 재고 상태 표기 (여유/부족/고갈) 검증
==============================================================
목적: 모니터링 화면에서 재고 상태가 올바르게 여유/부족/고갈로
      구분되어 표시되는지 확인

재고 상태 정의:
  여유 : 주문 대비 재고가 충분한 상태
  부족 : 주문 대비 재고 수량이 부족한 상태
  고갈 : 현재 재고 수량이 0인 상태

--------------------------------------------------------------
시나리오 A — 재고 여유 확인
--------------------------------------------------------------
STEP A1. 재고가 충분한 시료 확인
  메인 메뉴 → 3 (모니터링) → 재고량 확인

  기대 결과 (초기 상태):
    - S005 MEMS 압력센서: 재고 100개 → 상태: 여유

--------------------------------------------------------------
시나리오 B — 재고 부족 상태 만들기
--------------------------------------------------------------
STEP B1. 재고보다 많은 수량으로 주문 접수 + 승인
  주문 접수:
    시료 ID : S004
    고객명  : 부족테스트
    수량    : 25   (현재 재고 20보다 많음)

  주문 승인:
    → 상태 PRODUCING으로 전환 (재고 부족 확인)

STEP B2. 모니터링 재고 확인
  메인 메뉴 → 3 (모니터링) → 재고량 확인

  기대 결과:
    - S004 SiC 파워 소자: 상태: 부족 표시

--------------------------------------------------------------
시나리오 C — 재고 고갈 상태 만들기
--------------------------------------------------------------
STEP C1. 재고를 정확히 소진하는 주문
  (재고가 0이 되도록 주문 접수 + 승인)

  예시:
    시료 ID : S004
    수량    : 현재 남은 재고 수량 만큼

  승인 후 재고 = 0

STEP C2. 모니터링 재고 확인
  메인 메뉴 → 3 (모니터링) → 재고량 확인

  기대 결과:
    - S004 상태: 고갈 표시

--------------------------------------------------------------
판정 기준
  PASS: 여유/부족/고갈 상태가 정확히 구분되어 표시
  FAIL: 상태 표기가 실제 재고와 일치하지 않는 경우
