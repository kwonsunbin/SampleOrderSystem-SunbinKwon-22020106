==============================================================
TC07 | FIFO 생산 큐 순서 검증
==============================================================
목적: 생산 라인이 먼저 들어온 주문부터(FIFO) 처리하는지 확인

--------------------------------------------------------------
STEP 1. 프로그램 실행
--------------------------------------------------------------
  명령어: ./run.sh

--------------------------------------------------------------
STEP 2. 재고 부족 주문 3건 순서대로 접수 + 승인
--------------------------------------------------------------
  시료 S003 (GaN 에피택시 웨이퍼, 재고 15개) 사용

  주문1 접수:
    시료 ID : S003
    고객명  : 고객_FIRST
    수량    : 20   → 승인 → PRODUCING (주문 ID 메모: ID-1)

  주문2 접수:
    시료 ID : S003
    고객명  : 고객_SECOND
    수량    : 20   → 승인 → PRODUCING (주문 ID 메모: ID-2)

  주문3 접수:
    시료 ID : S003
    고객명  : 고객_THIRD
    수량    : 20   → 승인 → PRODUCING (주문 ID 메모: ID-3)

  ※ 각 주문 접수 후 바로 승인해야 순서가 보장됩니다.

--------------------------------------------------------------
STEP 3. 생산 큐 순서 확인
--------------------------------------------------------------
  메인 메뉴 → 4 (생산 라인 관리) → 1 (생산 큐 조회)

  기대 결과:
    1번째: 고객_FIRST  (ID-1)
    2번째: 고객_SECOND (ID-2)
    3번째: 고객_THIRD  (ID-3)
    → FIFO 순서로 나열

--------------------------------------------------------------
STEP 4. 첫 번째 처리
--------------------------------------------------------------
  메인 메뉴 → 4 (생산 라인 관리) → 2 (다음 주문 처리)

  기대 결과:
    - 처리 완료된 주문 ID가 ID-1 (고객_FIRST)

--------------------------------------------------------------
STEP 5. 두 번째 처리
--------------------------------------------------------------
  메인 메뉴 → 4 (생산 라인 관리) → 2 (다음 주문 처리)

  기대 결과:
    - 처리 완료된 주문 ID가 ID-2 (고객_SECOND)

--------------------------------------------------------------
STEP 6. 나머지 큐 확인
--------------------------------------------------------------
  메인 메뉴 → 4 (생산 라인 관리) → 1 (생산 큐 조회)

  기대 결과:
    - 고객_THIRD (ID-3) 1건만 남음

--------------------------------------------------------------
판정 기준
  PASS: 접수 순서대로 처리 (FIFO 보장)
  FAIL: 순서가 뒤바뀌어 처리된 경우
