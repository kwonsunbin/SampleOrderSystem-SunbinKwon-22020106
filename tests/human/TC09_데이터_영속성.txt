==============================================================
TC09 | 데이터 영속성 — 재시작 후 데이터 유지 확인
==============================================================
목적: 프로그램을 종료하고 재시작해도 주문 및 재고 데이터가
      그대로 유지되는지 확인 (JSON 파일 영속성)

--------------------------------------------------------------
STEP 1. 프로그램 실행 및 주문 접수
--------------------------------------------------------------
  명령어: ./run.sh

  주문 접수:
    시료 ID : S005
    고객명  : 영속성테스트
    수량    : 5

  주문 승인 → CONFIRMED
  주문 ID 메모 (화면에 표시된 UUID)

--------------------------------------------------------------
STEP 2. 재고 상태 메모
--------------------------------------------------------------
  메인 메뉴 → 1 (시료 관리) → 2 (시료 조회)

  현재 S005 재고 수량 메모: _____ 개

--------------------------------------------------------------
STEP 3. 프로그램 종료
--------------------------------------------------------------
  메인 메뉴에서 0 입력 (종료)

--------------------------------------------------------------
STEP 4. JSON 파일 직접 확인 (터미널)
--------------------------------------------------------------
  cat data/orders.json
    → 위 주문 ID가 status: "CONFIRMED" 로 저장되어 있어야 함

  cat data/samples.json
    → S005 stock이 STEP 2에서 메모한 값과 일치해야 함

--------------------------------------------------------------
STEP 5. 프로그램 재시작
--------------------------------------------------------------
  명령어: ./run.sh

--------------------------------------------------------------
STEP 6. 데이터 유지 확인
--------------------------------------------------------------
  메인 메뉴 → 3 (모니터링)
    → STEP 1에서 만든 주문이 CONFIRMED 상태로 표시되어야 함

  메인 메뉴 → 1 (시료 관리) → 2 (시료 조회)
    → S005 재고가 STEP 2에서 메모한 값과 동일해야 함

--------------------------------------------------------------
판정 기준
  PASS: 재시작 후 모든 데이터(주문, 재고) 그대로 유지
  FAIL: 재시작 후 데이터가 초기화되거나 다른 값으로 변경
